# RayMind 项目团队分工指南

## 项目结构

```
RayMind 智能农田机器人
│
├── 硬件层          ← 机械、硬件
├── 感知层          ← 传感器
├── 决策层          ← AI算法
├── 执行层          ← 运动控制
├── 通信层          ← ROS2、网络
└── 应用层          ← GUI、Web
```

---

## 推荐团队分工（3-5人）

### 方案一：3人团队

| 角色 | 人员 | 主要任务 |
|------|------|----------|
| 组长 | 1人 | 统筹协调、系统集成 |
| 硬件+感知 | 1人 | 传感器、机械 |
| 算法+软件 | 1人 | AI、控制、GUI |

### 方案二：5人团队

| 角色 | 人员 | 主要任务 |
|------|------|----------|
| 组长 | 1人 | 项目管理、系统集成 |
| 硬件工程师 | 1人 | 机械设计、硬件连接 |
| 感知工程师 | 1人 | 传感器、SLAM |
| 算法工程师 | 1人 | 杂草检测、路径规划 |
| 软件工程师 | 1人 | ROS2、GUI、通信 |

---

## 详细任务分解

### 1. 硬件工程师

| 任务 | 周期 | 交付物 |
|------|------|--------|
| 机械结构设计 | 2周 | 3D模型、零件清单 |
| 传感器选型 | 1周 | 选型报告 |
| 电路接线 | 1周 | 接线图 |
| 底盘组装 | 1周 | 整机 |
| 传感器安装 | 1周 | 装配完成 |

**技能要求**：
- 机械设计（SolidWorks）
- 电子电路
- PCB设计

---

### 2. 感知工程师

| 任务 | 周期 | 交付物 |
|------|------|--------|
| LiDAR驱动 | 1周 | ROS2驱动 |
| 相机标定 | 1周 | 标定文件 |
| IMU驱动 | 1周 | ROS2驱动 |
| GPS驱动 | 1周 | ROS2驱动 |
| 多传感器融合 | 2周 | SLAM方案 |

**技能要求**：
- ROS2
- OpenCV
- SLAM算法

---

### 3. 算法工程师

| 任务 | 周期 | 交付物 |
|------|------|--------|
| 杂草检测模型 | 3周 | YOLO模型 |
| 目标定位 | 2周 | 定位算法 |
| 路径规划 | 2周 | 覆盖路径算法 |
| 行为决策 | 2周 | 决策逻辑 |

**技能要求**：
- 深度学习（YOLO）
- 路径规划算法
- 强化学习

---

### 4. 软件工程师

| 任务 | 周期 | 交付物 |
|------|------|--------|
| ROS2架构 | 1周 | 节点设计 |
| 运动控制 | 2周 | 控制节点 |
| GUI开发 | 2周 | PyQt5界面 |
| Web监控 | 2周 | Web界面 |
| 通信模块 | 1周 | 网络配置 |

**技能要求**：
- Python/C++
- ROS2
- PyQt5/React

---

### 5. 组长/项目经理

| 任务 | 周期 | 交付物 |
|------|------|--------|
| 需求分析 | 1周 | 需求文档 |
| 进度跟踪 | 持续 | 周报 |
| 风险管控 | 持续 | 风险清单 |
| 集成测试 | 2周 | 测试报告 |
| 文档整理 | 1周 | 完整文档 |

**技能要求**：
- 项目管理
- 技术全面
- 文档能力

---

## 时间规划（总计12周）

```
第1-2周:  硬件准备 + 需求确认
第3-5周:  感知系统 + 硬件组装
第6-8周:  算法开发 + 控制实现
第9-10周: 软件集成 + GUI开发
第11周:   联调测试
第12周:   优化验收
```

## 代码管理

### Git分支策略

```
main        ← 稳定版
├── dev     ← 开发版
│   ├── hardware/xxx
│   ├── perception/xxx
│   ├── algorithm/xxx
│   └── software/xxx
└── release/1.0
```

### 提交规范

```
feat: 添加新功能
fix: 修复bug
docs: 文档更新
test: 测试代码
refactor: 重构
```
